Bienvenido

Obtén información sobre cómo desarrollar documentos para JW Library.


Esta página contienen guías e información de referencia para los desarrolladores que desean crear documentos para JWPUB Catalog de JW Library. Gracias por ser miembro de la comunidad de desarrolladores de JWPUB Catalog. ¡Nos alegra que estés aquí!


Antes de empezar

Conocimientos básicos de informática, tener instalado el software básico y conocimientos básicos de cómo trabajar con archivos. La Instalación de software básico, te muestra las herramientas que necesitas para hacer el desarrollo web simple, y la forma de instalarlas correctamente. Un documento JWPUB consta de muchos archivos: texto del contenido, código, contenido multimedia, etc. Cuando estás creando un documento, necesitas ensamblar estos archivos en una estructura sensible en tu computadora local, asegurarte de que puedan comunicarse entre sí y hacer que todo su contenido se vea bien antes de que eventualmente los cargues en un servidor. El manejo de archivos analiza algunos problemas que debes tener en cuenta, para que puedas configurar una estructura de archivos adecuada para tu documento. ¡Relajate! El objetivo es: familiarizarte con el lenguaje HTML, y adquirir algo de práctica escribiendo unos pocos elementos HTML en tu documento.

En esta sección vamos a exponer lo más básico del HTML. Para comenzar definiremos elementos, atributos y el resto de términos importantes que quizá ya te suenen y qué función cumplen dentro del lenguaje. También explica dónde encajan estos en HTML. Aprenderás cómo se estructuran los elementos HTML, cómo se estructura una página HTML típica y otras características básicas importantes del lenguaje. Por el camino, también iremos practicando con algo de HTML.

¿Qué herramientas necesitas?

Una computadora. Tal vez esto suena obvio para algunas personas, pero habrá quien esté leyendo este artículo desde el móvil o una computadora de biblioteca. Para el desarrollo web serio, es mejor invertir en un equipo de escritorio o portátil con Windows, Mac o Linux.

Un editor de texto, para escribir código. Puedes usar un editor de texto libre (ej. Brackets, Atom, Notepad++, Sublime Text, GNU Emacs, VIM, Visual Studio Code, WebStorm), no uses un editor híbrido. Los editores de documentos de oficina no son adecuados para esto, pues dependen de elementos ocultos que interfieren con los motores de renderizado usados por los navegadores.

Navegadores web, para probar el código. Actualmente los navegadores más usados son Firefox, Chrome, Opera, Safari, Vivaldi, Internet Explorer y Microsoft Edge. También debes comprobar cómo funciona tu documento en dispositivos móviles.

Esto parece una lista espeluznante pero, afortunadamente, puedes comenzar a trabajar en el desarrollo web sin saber nada de la mayoría de estas herramientas. En este artículo solo tendrás que configurar lo mínimo: un editor de texto y algunos navegadores web modernos.

Probablemente ya tengas un editor de texto básico instalado en tu computadora. De manera predeterminada Windows incluye el Bloc de notas y OS X viene con TextEdit. Las distros (versiones) de Linux varían: Ubuntu viene con Gedit; distribuciones basadas en KDE suelen traer Kate o Kwrite.

Para el desarrollo Web, probablemente hay cosas mejores que el Bloc de notas o TextEdit. Una recomendación puede ser empezar con Brackets, un editor gratuito que ofrece vistas previas en vivo y sugerencias de código.


Empezar

Las documentos están hechos de componentes diferentes, pero cohesivos. Los archivos JWPUB se tratan de una página de opciones, elementos de interfaz de usuario y varios archivos lógicos. Los componentes de un documento se crean con tecnologías de desarrollo web: HTML. Los componentes de un documento dependerán de su funcionalidad y es posible que no requieran todas las opciones.

Este tutorial creará un documento. Utilizará muchos de los componentes de la plataforma de componentes para dar una demostración introductoria de sus relaciones. La plataforma de componentes, o JWPUB Framework: es el marco HTML moderno que simplemente hará funcionar sus documentos. JWPUB Framework es un marco gratuito de código que proporciona componentes listos para usar que puede combinar fácilmente para crear interfaces web receptivas.

Para comenzar, cree un nuevo directorio para contener los archivos de la extensión.


Crear el manifiesto

Los documentos comienzan con su manifiesto. Cree un archivo al que se llame manifest.json e incluya el código siguiente:

{
  "title": "Programa de actividades para la adoración en familia de octubre",
  "displayTitle": "Adoración en familia",
  "publicationType": "Program",
  "password": Nothing,
  "language": "1",
  "imageFileName": "46616d696c7920576f726b73686973_univ_sqr.jpg:600:600",
  "htmlValidated": false,
  "version": "1",
  "expandedSize": "794398",
  "timestamp": "2022-09-11T11:59:26Z"
}

Analicemos este JSON anterior linea a linea. Los archivos en formato JSON, son utilizados por aplicaciones, y páginas web para guardar estructuras de datos simples que sean legibles en un formato de texto normal. En el caso de JW Library, los archivos manifest.json, podríamos decir de forma sencilla que son la carta de presentación del archivo JWPUB que se instalará. Pues contienen datos informativos, entre ellos, el tipo de publicación que contiene, el título, categoría, tamaño, firmas, etc. title y displayTitle serán los títulos completos y abreviados que aparecerán en JWPUB Catalog. Por favor respeta el siguiente formato una vez que hagas tu proyecto. publicationType esto es el tipo de publicación en este caso “Programa”. language aquí va el número de lenguaje correspondiente al idioma español, en este caso es 1. imageFileName es el nombre del archivo en formato JPG que aparecerá en JWPUB Catalog, y no olvides agregar los valores de tamaño. :600:600, claro, estos valores dependerán de tu archivo. expandedSize es el tamaño del archivo *.htm en bytes. Este campo se refiere al tamaño que ocupa la publicación. Para obtener este valor, puedes ver el tamaño del archivo, haciendo clic derecho sobre este y viendo las propiedades. Copiando el tamaño en bytes, asignarás un valor exacto del tamaño de tu publicación. timestamp es la fecha y hora de creación en la publicación. Formato Año/Mes/Día


publicationType

Brochures                   Folletos
